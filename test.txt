1. Проверка стандартного запроса
curl -i http://localhost:8080

Ожидаемый результат:

Приложение возвращает JSON-ответ, поле X-Forwarded-For содержит цепочку IP-адресов
В цепочке присутствуют: IP клиента, IP всех nginx-прокси по пути запроса

2. Проверка передачи цепочки через несколько прокси
curl -i http://localhost:8080

Ожидаемый результат:

X-Forwarded-For содержит несколько IP-адресов, разделённых запятыми, порядок соответствует маршруту прохождения запроса:
клиент → nginx-edge → nginx-internal-1 → nginx-internal-2

3. Проверка защиты от подмены X-Forwarded-For (простая подделка)
curl -i http://localhost:8080 \
  -H "X-Forwarded-For: 8.8.8.8"

Ожидаемый результат:

Значение 8.8.8.8 отсутствует в ответе приложения, используется реальный IP клиента, заголовок, переданный пользователем, игнорируется

4. Проверка защиты от сложной подмены заголовка
curl -i http://localhost:8080 \
  -H "X-Forwarded-For: evil.com, 1.1.1.1"

Ожидаемый результат:

Все пользовательские значения полностью игнорируются, в ответе присутствует только доверенная цепочка прокси, отсутствуют любые значения, переданные клиентом

5. Проверка стабильности цепочки
curl -i http://localhost:8080
curl -i http://localhost:8080
curl -i http://localhost:8080

Ожидаемый результат:

Цепочка X-Forwarded-For формируется одинаково для каждого запроса, порядок IP-адресов остаётся стабильным, отсутствуют пропуски или дублирование узлов
